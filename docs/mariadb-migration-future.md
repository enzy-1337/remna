# Переход на MariaDB (будущая задача)

Сейчас проект **целиком на PostgreSQL** (`asyncpg`, `DATABASE_URL=postgresql+asyncpg://…`). Этот файл — чеклист на следующий заход, когда решите переносить прод или поднимать MariaDB.

## Текущее состояние

- Движок: `shared/database.py` → `create_async_engine(settings.database_url, …)`.
- Миграции: Alembic, ревизии в `migrations/versions/` (в т.ч. JSON-колонки, `DateTime(timezone=True)`, `UUID`, частичные индексы).
- Docker: `docker-compose.yml` — сервис `postgres:16-alpine`.

Заглушка в коде: `shared/db_mariadb_future.py` (напоминание, что отдельный бэкенд не подключён).

## Зачем отдельный документ

MariaDB **5.5.x** сильно отличается от PostgreSQL 16: другой диалект SQL, ограничения по JSON, типам, индексам и поведению миграций. «Просто сменить URL» недостаточно — нужен проход по схеме и тестам.

## Чеклист миграции (когда возьмётесь)

1. **Версия MariaDB**  
   По возможности планируйте **не ниже 10.5+** (нормальный JSON, совместимость с SQLAlchemy проще). Если жёстко нужна 5.5 — отдельно оцените каждую миграцию (JSON может жить в `TEXT`, нет нативного `JSONB` и т.д.).

2. **Async-драйвер**  
   Подобрать стек под SQLAlchemy 2 async, например `asyncmy` / `aiomysql` + корректный префикс URL для `create_async_engine` (документация SQLAlchemy для выбранного диалекта).

3. **`DATABASE_URL`**  
   Вынести в `.env` формулировку вида `mysql+asyncmy://user:pass@host:3306/db?charset=utf8mb4` (точный префикс — по выбранному драйверу). Убедиться в `utf8mb4` для эмодзи в Telegram-полях.

4. **Alembic**  
   - В `migrations/env.py` и `alembic.ini` — диалект `mysql` / `mariadb` вместо `postgresql`.  
   - Прогнать `alembic upgrade head` на **чистой** тестовой БД.  
   - Исправить несовместимые операции (если Alembic сгенерировал под PG-специфику — ручные правки или новые ревизии).

5. **Типы и конструкции**  
   Пройтись по моделям в `shared/models/` и ревизиям:
   - `JSON` / `JSONB` → совместимый тип под вашу версию MariaDB.
   - `UUID` — как хранить (BINARY(16), CHAR(36), или оставить строки).
   - `DateTime(timezone=True)` — поведение на MariaDB.
   - Частичные/уникальные индексы из `0002` и др. — проверить синтаксис.

6. **Бэкапы**  
   Скрипты бэкапа сейчас ориентированы на `pg_dump` (`shared/services/backup_service.py` и переменные `BACKUP_*`). Для MariaDB понадобится `mysqldump` (или аналог) и отдельный путь в том же сервисе/кроне.

7. **Регрессия**  
   - `pytest`  
   - Ручной смоук: регистрация, пополнение, вебхуки, биллинг v2, тикеты.

## Вебхуки Remnawave (нет образцов payload)

Когда появятся реальные JSON от панели, сохраните 2–3 файла в репозиторий (например `tests/fixtures/remnawave_webhook_traffic.json`, `…_hwid.json`) и обновите парсер в `shared/services/billing_v2/webhook_ingress_service.py` + тесты в `tests/test_webhook_processing.py`. До этого ориентируйтесь на логи таблицы `remnawave_webhook_events` в проде/стейдже.

## Реферальный процент

Процент `REFERRAL_TOPUP_PERCENT` начисляется **только с пополнений баланса** (вебхуки Platega/CryptoBot и т.п.), а не с списания за тариф с баланса. Отдельно остаётся логика «первая платная покупка тарифа» (`REFERRAL_INVITER_REWARD_*`), если она включена в конфиге.
